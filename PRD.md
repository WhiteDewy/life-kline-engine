## 1. 产品定位

### 1.1 产品名称
**星灵花园** (Star Spirit Garden) —— 原"占星人生模型"

### 1.2 一句话
**以星辰为据，与心对话。** —— 一个有理有据的陪伴疗愈产品，以星灵和星座为镜，帮助用户与自己内心对话。

### 1.3 产品类型
陪伴疗愈型产品。不是工具，不是算命，不是内容平台。
- **入口体验**：情绪驱动（"和我聊聊"），不是功能驱动（"生成报告"）
- **核心循环**：与星灵对话 → 看见自己 → 星灵日记积累 → 更深的内省
- **信任基础**：古典+现代占星引擎提供数据事实层，AI 在事实层之上做陪伴表达
- **差异化**：22 位星灵/座灵 = 22 面内心之镜，不是通用情绪抚慰，是专属星盘驱动的个性化回应

### 1.4 三条护城河
1. **星盘数据**：每个用户的星盘是唯一的。星灵的每一句话都有行星度数、宫位、相位的依据。
2. **星座知识体系**：古典+现代占星双轨，庙旺失陷、法达周期、飞星链路——AI 无法凭空编造的结构性知识。
3. **星灵日记**：对话提取关键词 → 自动生成日记 → 时间积累 → 模式识别。存得越久，迁移成本越高。

### 1.5 三层架构

### 1.3 三层架构

```
引擎层 →  占星计算 + 规则判断 → 多角度解释矩阵（锚点，不可推翻）
AI 层  →  对话翻译 + 角度切换  → 根据用户反馈在矩阵内调整
前端层 →  领域展示 + 对话入口  → 用户看到的叙事 + 隐藏的调试面板
```

## 2. 目标用户与用户问题

### 2.1 用户画像
一个迷茫的人，ta 不关心占星流派和技术名词，只想知道自己的真实答案。

### 2.2 14 个用户问题与领域映射

```

  2. 别人怎么看我？我的外表和气质给人什么感觉？  → 外形气质

  4. 我的工作能力怎么样？什么环境适合我？       → 工作技能

  6. 我的钱从哪里来？正财稳还是偏财好？         → 财务格局

  7. 我为什么总是在感情里遇到同样的问题？       → 桃花感情
  8. 我适合什么样的伴侣？婚姻会怎样？           → 婚姻画像
  9. 我和父母的关系为什么是这样？               → 原生家庭

```

## 3. 引擎核心原则

### 3.1 基础原则
- **本命定义底盘**：本命盘是人生的地图和车辆型号，不与时间法平级竞争
- **宫性优先、星性修饰**：占星分析以宫位为第一层，行星为第二层，星座为第三层
- **古现分工**：古占负责"这件事容不容易成"（事件/结构），现占负责"你在心理上怎么处理这件事"（动机/感受）
- **引擎锚点**：引擎的所有结构性判断是锚点，不可被推翻。用户反馈"不准"时是取象角度需要切换
- **多角度解释矩阵**：引擎输出同一组数据的多个取象角度，AI 在矩阵内切换，不发明新信息
- **数值内部化**：所有评分保留在引擎内部，用户侧只输出叙事段落
- **引擎即核心**：引擎及其知识库（`interpretation/*`、`domains/*`、`constants.py` 中的行星原型/宫位含义/星座人格等结构化知识）是所有叙事的唯一来源。前端 UI 的叙事风格不得绕过引擎知识库在前端硬编码占星解读文案。

### 3.2 飞星原则
- **宫性优先**：飞星的核心议题由目标宫位（飞到哪）决定
- **星性修饰**：同一目标宫位，不同来源宫主星带来不同表达方式（planet_mod 模板引擎）

### 3.3 解释层级
```
【结构结论】 ← 古占为主（事件潜力和结构）
【心理感受】 ← 现占为主（内在动机和体验）
【建议方向】 ← 两者综合（现实行动指南）
```

### 3.4 禁止事项
- 不贩售恐惧，不制造依赖
- 不做宿命断言或预测性结论
- 不针对特定人物做硬编码分支
- 不暴露数值给用户
- **不在前端硬编码占星解读文案**：所有面向用户的叙事段落（标题、正文、建议、运势偏向等）必须由引擎知识库规则生成，前端不得绕过引擎自行编写宫位/行星/尊严/趋势相关的解读文本

## 5. 时间法体系

### 5.1 设计原则
- 按时间粒度组织，不按技术流派组织
- 法达定调（一生）、日返定年（全景）、月返定月（外在）、三限定月（内在）
- 日弧和行运作为事件节点叠加，不做独立模块

### 5.2 七项时间法

| 技术 | 角色 | 时间跨度 | 状态 |
|:---|:---|:---|:---|
| 法达 (Firdaria) | 一生定调。大运+子运。基于星盘的100年结构基线（不是算命） | 一生 | ✅ |
| 日弧 (Solar Arc) | 跨年事件节点。叠加在法达时间线上，标记转折区间（±1年容许度） | 跨年 | 待完善 |
| 日返 (Solar Return) | 年度全景。生日到生日，年度运势底色 | 1年 | 待完善 |
| 次限 (Secondary Progression) | 年度内在演化。次限月亮2.5年换座——心理演化的慢变量 | 数年 | 待完善 |
| 月返 (Lunar Return) | 月度外在指引。月亮回到本命位置，本月外界氛围和事件倾向 | 1月 | ✅ |
| 三限 (Tertiary Progression) | 月度内在节律。次限的加速版——本月内心在演化什么。连续趋势能回答"什么时候这个感受会过去" | 1月 | 待完善 |
| 行运 (Transits) | 即时触发提醒。快星叠加月返做本月提醒，慢星叠加法达做长期背景 | 天-月 | ⚠️ 引擎完成，前端集成中 |

### 5.3 已放弃的技术
- **小限 (Profection)**：只告诉你"今年第几宫被激活"，但激活后发生什么需要其他技术补充。作为用户产品多一层翻译，边际收益低。放弃。

## 6. 10 行星人格维度系统 ← v2.1 更新

### 6.1 设计理念

借鉴**万象有灵**的十神人格维度系统，将 **10 颗行星**（传统 7 + 外行星 3）定义为 10 个**人格维度**。

- **太阳 = 主人格**（始终可见，代表核心自我）
- **月亮/水星/金星/火星/木星/土星/天王星/海王星/冥王星 = 次人格**
- **三层塑造**：行星本性（做什么）→ 星座风格（怎么做）→ 宫位舞台（在哪做）

```
十神（万象有灵）  →  十行星（本系统 v2.1）
10 个虚拟人格维度  →  10 颗行星 = 10 个人格维度
太阳 → 主人格      →  常驻主角
月金水火木土天海冥 → 次人格 → 按激活度轮换
```

> **v1.5 遗留**: 12 星座角色系统（原 §6.1-6.6）保留为兼容模式，通过 `🎭 星座角色` tab 可访问。v2.0 的 `🌌 行星人格` 是推荐入口。

### 6.3 角色个性化

从用户星盘计算每个行星角色的**个性化参数**：

- **core_strength (0-100)**: 尊贵状态(40%) + 宫位力量(30%) + 命主星(15%) + 发光体(15%)
- **sign_flavor**: 行星落入的星座风格层（element, modality, voice_tone）
- **house_context**: 行星落入的宫位领域层（domain, topic）
- **dignity_label**: 入庙/旺相/平/失势/落陷
- **role_tag**: 天赋维度 / 人生课题 / 核心表达 / 活跃影响 / 背景影响

### 6.4 每日唤醒

每日根据时间律动计算 7 行星的**激活度** (0-100)：

| 因子 | 权重 |
|------|------|
| 法达主运星 = 该行星 | +30 |
| 法达子运星 = 该行星 | +15 |
| 行运合相该行星 | +12 |
| 当前月亮星座 = 该行星落座 | +20 |
| core_strength >= 60 | +5 |

太阳始终以 80 分保底激活。每日选出 top 2-3 作为"今日副人格"。

### 6.3 角色个性化

从用户星盘计算每个角色的**个性化参数**：

- **存在感 (presence)**：该星座在星盘中的表达强度。上升/太阳/月亮星座 + 行星落点 + 宫头落点 + 星群 bonus → 归一化 0-100
- **舒适度 (comfort)**：落此星座的行星尊贵状态。庙旺=舒适 → 正分，失势落陷=不适 → 负分
- **角色标签**：存在感≥50+舒适≥10="天赋角色"，存在感≥50+舒适≤-10="课题角色"，存在感<20="背景角色"
- **故事线**：飞星链路中涉及该星座宫头的路径
- **领域关联**：该星座覆盖的宫位对应的12个生命领域

### 6.4 每日唤醒

每天根据时间律动计算 12 角色的**激活度** (0-100)：

| 因子 | 权重 |
|------|------|
| 法达主运星所在星座 | +30 |
| 法达子运星所在星座 | +15 |
| 当前月亮星座 | +25 |
| 当前太阳星座 | +20 |
| 行运行星触发 | +5~15/颗 |
| 本命存在感≥50 | +5 |

每日选出前 3 名作为"今日登场角色"。

### 6.5 多视角内容

- **单角色对话** (`POST /api/characters/{id}/chat`)
- **角色议会** (`POST /api/characters/{id}/council`)：多角色对同一话题给出不同视角
- **每日简报** (`GET /api/characters/{id}/daily`)

### 6.6 成长追踪

追踪用户互动形成动态画像：
- 对话历史 (per 角色)
- 话题偏好
- 里程碑 (首次对话/10次对话/连续7天)
- 亲密度 (0-10 scale)

存储位置：`backend/data/{report_id}_growth.json`

## 10. 星灵花园——陪伴疗愈产品设计

### 9.1 产品定位

**产品名**：星灵花园 (Star Spirit Garden)

**一句话**：以星辰为据，与心对话。

**产品类型**：陪伴疗愈型。22 位星灵（10 行星灵 + 12 星座灵）作为用户内在原型的拟人化投射，通过 AI 对话实现自我内省和情绪陪伴。

**核心主张**：
- 星灵即心镜——用户和火星灵聊天，本质上是在面对自己的愤怒与勇气
- 舒服是入口，信服是留存——AI 对话是"糖衣"，占星引擎是"炮弹"
- 不贩卖恐惧，不制造依赖，只做陪伴和映照

**对标万象有灵的差异**：
| 万象有灵 | 星灵花园 |

| 10 神（八字十神） | 22 灵（10 行星 + 12 星座），多一个维度 |

| 无星盘数据 | 以用户本命星盘为个性化基础 |
| 无日记系统 | 星灵日记——对话→关键词提取→自动生成→长期模式识别 |

```

    ↓

**日记的长期价值**：
- 第1天：新鲜感，"记下来了"
- 第7天：模式初现，"我好像每周三最焦虑"
- 第30天：深度洞察，"我聊月亮灵12次，关键词是'安全感'"

## 11. 引擎占星师——规则驱动的占星咨询核心 ← v2.1 新增

### 11.1 定位

引擎占星师是**三层咨询架构的底座**。不依赖 AI/LLM，确定性输出。AI 层是后续的"翻译美化"层，引擎层永远在线。

```
引擎占星师 = 规则驱动的占星咨询推理核心
AI 占星师   = LLM 拿到引擎结构化输出后做翻译美化（消耗 token）
真人占星师  = 真人咨询，基于引擎报告做深度对话（后续接入）
三者独立互补，用户自选。
```

### 11.2 核心逻辑：宫性大于星性

```
宫位 > 宫主星 > 宫内星 > 相位 > 接纳互溶

宫内星 = 过程（你怎么经历这个领域——体验、感受、行为模式）
宫主星 = 结果（这个领域最终怎么收场——走向、结果、终点）
```

例子：
> 问事业 → 10宫主是土星 → 土星飞6宫 → 事业靠日常专业积累  
> 宫内星火星（过程：工作中有冲劲、急性子）  
> 宫主星土星（结果：长期积累才能出成绩，急不来）

### 11.3 三层架构

```
用户消息
  ↓
IntentRouter    → 关键词匹配 → domain + 情绪状态
  ↓
ChartReader     → 收集结构化星盘证据：
                  · 落宫落座（过程层）
                  · 掌宫·飞星（结果层）
                  · 相位·接纳互溶（影响层）
  ↓
VoiceRenderer   → 参数化模板 → 三段式对话：
                  承接（回应用户话语）
                    → 镜映（过程→结果→影响 两段式）
                      → 引导（行星 persona 建议风格）
  ↓
EngineResponse  → {full_text, evidence, ...}
```

### 11.4 各组件定义

**IntentRouter**：9 领域关键词映射 + 5 种情绪检测（焦虑/迷茫/沮丧/期待/好奇）。topic_hint 可覆盖弱匹配。无匹配时路由到 personal。

**ChartReader**：从 report_data 中采集 6 类结构化证据——
- `read_planet_profile()`：落宫落座尊贵
- `read_rulerships()`：该行星掌管哪些宫位
- `read_flystars()`：宫主星飞入的宫位
- `read_aspects()`：相位信息（吉/凶/合）
- `read_receptions()`：接纳与互溶
- `collect_structured_reading()`：以上六类汇总

**VoiceRenderer**：参数化模板引擎，非穷举。按 `{sign_label}` `{house}` `{core_theme}` `{dignity_note}` 等变量填充。10 行星各有镜映模板和引导模板。

### 11.5 12 宫位增强定义

每个宫位从原来的 1 词标签扩展为：

| 字段 | 说明 |
|------|------|
| `title` | 中文标签 |
| `domain_key` | 对应的领域 key |
| `keywords` | 关键词列表 |
| `adult` | 成人社会义 |
| `power` | 权力动力义 |
| `ruler_meaning` | **宫主星导向（结果层）** —— 宫主星落宫告诉你的信息 |
| `planet_in_meaning` | **宫内星导向（过程层）** —— 宫内星告诉你的信息 |

宫位→领域映射表（`HOUSE_TO_DOMAIN`）统一维护在 `interpretation/house_rules.py`。

### 11.6 星灵对话权限

引擎占星师不是完全免费：
