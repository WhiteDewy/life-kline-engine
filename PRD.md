## 1. 产品定位

### 1.1 产品名称
占星人生模型（Astrology Product System）

### 1.2 一句话
一个占星解释器——用结构性的星盘分析，回答普通人对人生的真实疑问。

### 1.3 三层架构

```
引擎层 →  占星计算 + 规则判断 → 多角度解释矩阵（锚点，不可推翻）
AI 层  →  对话翻译 + 角度切换  → 根据用户反馈在矩阵内调整
前端层 →  领域展示 + 对话入口  → 用户看到的叙事 + 隐藏的调试面板
```

## 2. 目标用户与用户问题

### 2.1 用户画像
一个迷茫的人，ta 不关心占星流派和技术名词，只想知道自己的真实答案。

### 2.2 14 个用户问题与领域映射

```
认识自己
  1. 我是什么样的性格？我的喜好与做事风格，我有什么优势和盲区？      → 人格底色
  2. 别人怎么看我？我的外表和气质给人什么感觉？  → 外形气质

事业与财富
  3. 我适合做什么事业？方向在哪里？            → 事业方向
  4. 我的工作能力怎么样？什么环境适合我？       → 工作技能
  5. 我应该继续读书还是工作？适合学什么？       → 学业方向
  6. 我的钱从哪里来？正财稳还是偏财好？         → 财务格局

关系与情感
  7. 我为什么总是在感情里遇到同样的问题？       → 桃花感情
  8. 我适合什么样的伴侣？婚姻会怎样？           → 婚姻画像
  9. 我和父母的关系为什么是这样？               → 原生家庭
  10. 我适合找什么样的人合伙做事？              → 事业合伙
  11. 我和孩子的关系怎么处理？                  → 亲子关系

当下指引
  12. 我现在为什么这么痛苦？什么时候能好？       → 当前阶段
  13. 这个月我应该抓住什么、避开什么？           → 月度指引
  14. 今年我运势怎么样？                        → 年度指引

身体
  15. 我天生体质怎么样？容易在哪里出问题？       → 健康体质
```

## 3. 引擎核心原则

### 3.1 基础原则
- **本命定义底盘**：本命盘是人生的地图和车辆型号，不与时间法平级竞争
- **宫性优先、星性修饰**：占星分析以宫位为第一层，行星为第二层，星座为第三层
- **古现分工**：古占负责"这件事容不容易成"（事件/结构），现占负责"你在心理上怎么处理这件事"（动机/感受）
- **引擎锚点**：引擎的所有结构性判断是锚点，不可被推翻。用户反馈"不准"时是取象角度需要切换
- **多角度解释矩阵**：引擎输出同一组数据的多个取象角度，AI 在矩阵内切换，不发明新信息
- **数值内部化**：所有评分保留在引擎内部，用户侧只输出叙事段落
- **引擎即核心**：引擎及其知识库（`interpretation/*`、`domains/*`、`constants.py` 中的行星原型/宫位含义/星座人格等结构化知识）是所有叙事的唯一来源。前端 UI 的叙事风格不得绕过引擎知识库在前端硬编码占星解读文案。

### 3.2 飞星原则
- **宫性优先**：飞星的核心议题由目标宫位（飞到哪）决定
- **星性修饰**：同一目标宫位，不同来源宫主星带来不同表达方式（planet_mod 模板引擎）

### 3.3 解释层级
```
【结构结论】 ← 古占为主（事件潜力和结构）
【心理感受】 ← 现占为主（内在动机和体验）
【建议方向】 ← 两者综合（现实行动指南）
```

### 3.4 禁止事项
- 不贩售恐惧，不制造依赖
- 不做宿命断言或预测性结论
- 不针对特定人物做硬编码分支
- 不暴露数值给用户
- **不在前端硬编码占星解读文案**：所有面向用户的叙事段落（标题、正文、建议、运势偏向等）必须由引擎知识库规则生成，前端不得绕过引擎自行编写宫位/行星/尊严/趋势相关的解读文本

### 4.2 各领域核心指标

**人格底色 (personal, 0.5)**
- 古占：上升 + 命主星庙旺失陷 + 昼夜派系 + 飞星两步链路
- 现占：日月升三巨头 + 主导行星 + 相位网络

**外形气质 (appearance, 0.5)**
- 上升星座 + 命主星状态 + 1宫宫内星 + 上升紧密相位

**事业方向 (career, 0.7)**
- 古占：10宫主星庙旺、飞星两步链路、与1/8宫主关系
- 现占：MC + 10宫行星 + 太阳-10宫相位

**工作技能 (work_skill, 0.5)**
- 古占：6宫主星状态、飞星、与水星关系
- 现占：6宫行星 + 水星状态

**学业方向 (education, 0.5)**
- 古占：3/9宫主星、3-9对宫、水星+木星状态
- 现占：水星-木星相位

**财务格局 (finance, 0.8)**
- 古占：2/5/8/11四财宫、2-5-8-11飞星三角、接纳互溶
- 现占：金星-木星-冥王星相位

**桃花感情 (romance, 0.4)**
- 古占：5/7宫主星、金星-火星相位
- 现占：金星-火星、金星-木星/天王星/海王星/冥王星

**婚姻画像 (marriage, 0.7)**
- 古占：7宫主星庙旺失陷、飞星、与1/10宫主关系
- 现占：下降点 + 7宫行星

**原生家庭 (family, 0.7)**
- 古占唯一标准：4宫主=父亲，10宫主=母亲。日月作辅助参考
- 现占：4-10对宫情感残留

**事业合伙 (partnership, 0.8)**
- 古占：7宫主+10宫主关系、土星-7宫主相位、1-7权力博弈
- 与婚姻共享7宫计算，但叙事模板不同——侧重能力互补和利益结构

**亲子关系 (children, 0.5)**
- 古占：5宫主星、月亮（养育方式）、1-5宫主相位
- 现占：内在小孩投射

**健康体质 (health, 0.7)**
- 古占：1宫（先天体质）+ 6宫（日常健康）+ 8/12宫（慢性/隐性）
- 行星角色：火星=急性，土星=慢性，水星=神经，月亮=消化，海王=免疫

## 5. 时间法体系

### 5.1 设计原则
- 按时间粒度组织，不按技术流派组织
- 法达定调（一生）、日返定年（全景）、月返定月（外在）、三限定月（内在）
- 日弧和行运作为事件节点叠加，不做独立模块

### 5.2 七项时间法

| 技术 | 角色 | 时间跨度 | 状态 |
|:---|:---|:---|:---|
| 法达 (Firdaria) | 一生定调。大运+子运。基于星盘的100年结构基线（不是算命） | 一生 | ✅ |
| 日弧 (Solar Arc) | 跨年事件节点。叠加在法达时间线上，标记转折区间（±1年容许度） | 跨年 | 待完善 |
| 日返 (Solar Return) | 年度全景。生日到生日，年度运势底色 | 1年 | 待完善 |
| 次限 (Secondary Progression) | 年度内在演化。次限月亮2.5年换座——心理演化的慢变量 | 数年 | 待完善 |
| 月返 (Lunar Return) | 月度外在指引。月亮回到本命位置，本月外界氛围和事件倾向 | 1月 | ✅ |
| 三限 (Tertiary Progression) | 月度内在节律。次限的加速版——本月内心在演化什么。连续趋势能回答"什么时候这个感受会过去" | 1月 | 待完善 |
| 行运 (Transits) | 即时触发提醒。快星叠加月返做本月提醒，慢星叠加法达做长期背景 | 天-月 | ⚠️ 引擎完成，前端集成中 |

### 5.3 已放弃的技术
- **小限 (Profection)**：只告诉你"今年第几宫被激活"，但激活后发生什么需要其他技术补充。作为用户产品多一层翻译，边际收益低。放弃。

## 6. 12 星座人格角色系统 ← v1.5 新增

### 6.1 设计理念

借鉴**万象有灵**的十神人格维度系统，将 12 星座定义为 12 个**人格化角色**。每个星座是一个有性格、有声音、有记忆的 AI 角色。用户不是阅读报告，而是与角色**对话**。

```
十神（万象有灵）  →  十二星座（本系统）
10 个虚拟人格维度  →  12 个星座角色维度
每日唤醒          →  时间律动驱动的每日激活
```

### 6.2 12 角色原型

| 星座 | 角色原型 | 元素 | 守护星 | 一句话 |
|------|---------|------|--------|--------|
| 白羊座 | 战士/先锋 | 火 | 火星 | 敢为人先的行动者 |
| 金牛座 | 守护者/价值官 | 土 | 金星 | 稳得住的价值创造者 |
| 双子座 | 信使/连接者 | 风 | 水星 | 好奇心驱动的信息连接者 |
| 巨蟹座 | 照料者/滋养者 | 水 | 月亮 | 情感驱动的守护者 |
| 狮子座 | 王者/创造者 | 火 | 太阳 | 用创造力感染世界的主角 |
| 处女座 | 工匠/精进者 | 土 | 水星 | 用精确打磨一切的完美主义者 |
| 天秤座 | 外交官/平衡者 | 风 | 金星 | 在关系中寻找平衡的艺术大师 |
| 天蝎座 | 转化者/深度探索者 | 水 | 冥王星 | 在深渊中看见真相的转化者 |
| 射手座 | 探索者/传道者 | 火 | 木星 | 用信念引领方向的探索者 |
| 摩羯座 | 建筑师/责任者 | 土 | 土星 | 用耐心建立长期价值的建筑师 |
| 水瓶座 | 革新者/觉醒者 | 风 | 天王星 | 打破常规的系统思考者 |
| 双鱼座 | 诗人/融合者 | 水 | 海王星 | 在想象中融合一切的梦想家 |

### 6.3 角色个性化

从用户星盘计算每个角色的**个性化参数**：

- **存在感 (presence)**：该星座在星盘中的表达强度。上升/太阳/月亮星座 + 行星落点 + 宫头落点 + 星群 bonus → 归一化 0-100
- **舒适度 (comfort)**：落此星座的行星尊贵状态。庙旺=舒适 → 正分，失势落陷=不适 → 负分
- **角色标签**：存在感≥50+舒适≥10="天赋角色"，存在感≥50+舒适≤-10="课题角色"，存在感<20="背景角色"
- **故事线**：飞星链路中涉及该星座宫头的路径
- **领域关联**：该星座覆盖的宫位对应的12个生命领域

### 6.4 每日唤醒

每天根据时间律动计算 12 角色的**激活度** (0-100)：

| 因子 | 权重 |
|------|------|
| 法达主运星所在星座 | +30 |
| 法达子运星所在星座 | +15 |
| 当前月亮星座 | +25 |
| 当前太阳星座 | +20 |
| 行运行星触发 | +5~15/颗 |
| 本命存在感≥50 | +5 |

每日选出前 3 名作为"今日登场角色"。

### 6.5 多视角内容

- **单角色对话** (`POST /api/characters/{id}/chat`)
- **角色议会** (`POST /api/characters/{id}/council`)：多角色对同一话题给出不同视角
- **每日简报** (`GET /api/characters/{id}/daily`)

### 6.6 成长追踪

追踪用户互动形成动态画像：
- 对话历史 (per 角色)
- 话题偏好
- 里程碑 (首次对话/10次对话/连续7天)
- 亲密度 (0-10 scale)

存储位置：`backend/data/{report_id}_growth.json`

## 7. 页面架构

### 6.1 三页面结构

| 页面 | 路由 | 功能 |
|:---|:---|:---|
| 首页 | `/` | 分析目录。按"认识自己/事业与财富/关系与情感/当下指引/身体"分组展示入口 |
| 输入页 | `/analysis/:type` | 统一出生信息表单 |
| 报告页 | `/reports/:id` | 一张报告包含所有领域。Hero → 星盘可视化 → 领域 chip 选择器（单领域展开） → 时间导航区 → AI 对话入口 |

### 6.2 报告页内容顺序

```
Hero 区
  三段个性化观察（太阳/月亮/金星） + 行运高亮 + 法达阶段锚点 + 悬念钩子
  星盘速览条（上升 + 日月金火）

━━━ 星盘可视化 ━━━
  本命星盘图（NatalChartWheel） + 人生节律图（法达K线） + 当前行运面板

━━━ 领域探索 ━━━
  12 个 DomainPanel 以 chip 按钮横向排列
  点击 chip 展开/收起单个领域详情（同一时间仅展开一个）
  领域 chip 仅显示后端实际产出的领域（动态过滤）

  每个 DomainPanel 内部结构：
  - hero_bridge：承接 Hero 叙事
  - 核心议题一行
  - 结构结论段落
  - 心理层面（蓝色区块）
  - 建议方向（金色区块）
  - 底部对话钩子："聊聊XX"

━━━ AI 深度对话入口 ━━━
  CTA 卡片 → 引导用户进入对话模式（功能开发中）

━━━ 更多（规划中） ━━━
  法达时间表 · 隐藏调试入口 · 本命蓝图展开
```

### 6.3 DomainPanel 通用组件

DomainPanel 是领域内容的标准渲染组件。Props：`domain: DomainReport`。内部结构：
- `hero_bridge`：一句承接 Hero 叙事的过渡语
- `core_theme`：核心议题一行
- `structure`：结构结论（古现融合，按权重），按段落渲染，首段左侧金线强调
- `psychology`：心理层面（蓝色左边框区块，🧠 标签）
- `suggestion`：建议方向（金色边框区块，💡 标签）
- 底部 hook：分割线 + "是不是说到你心里了？" + "聊聊XX" 按钮

交互逻辑：12 个领域 chip 横向排列在领域探索区。点击 chip 展开对应的 DomainPanel（再次点击同一 chip 或点击其他 chip 切换）。同一时间仅展开一个领域。

### 6.4 隐藏调试入口（规划中）
连续点击页面 logo 5 次 → 展开 `_analysis_evidence` 面板。供占星学习者和开发者使用。

### 6.5 本命蓝图（后续优化）

后端已产出 `natal_blueprint` 和 `advanced_patterns` 数据，对应前端组件 `NatalBlueprintPanel.vue` / `AdvancedPatternsPanel.vue` 已开发完成。当前暂不接入报告页主流程，待收集领域分析的用户使用反馈后，确定放在 Hero 下方作为独立"本命蓝图"区块还是折叠式展开。
