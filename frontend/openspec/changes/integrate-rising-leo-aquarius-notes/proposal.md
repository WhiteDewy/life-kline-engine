# Change Proposal: integrate-rising-leo-aquarius-notes

## Why

当前项目已经有方法论面板、首页方法论区和本命报告页，但内容仍以通用原则为主，缺少可以直接体现咨询师风格的专题型教学资料。

`0613 占星笔记`里的“上升狮子 / 上升水瓶”材料有几个很强的价值：

- 它不是纯术语，而是把上升、太阳、月亮、命主星、伴侣主题、关系舒适度串成了咨询语境。
- 它能直接解释“别人最先看到你什么”“熟悉以后看到什么”“亲密关系里什么最重要”。
- 它既有面向普通用户的理解入口，也有面向专业占星师的判断提醒。

如果继续把这些内容停留在零散笔记里，项目会出现两个问题：

- 方法论区过于抽象，用户看完不知道这些原则如何落到真实人格和关系中。
- 报告页已经有星盘和结构，但缺少“如果你是某种上升，这张盘在现实里更像什么”的专题翻译层。

## What Changes

本次变更将把 `0613 占星笔记` 结构化后融入前端内容层，重点覆盖上升狮子与上升水瓶两个专题。

### Scope

1. 新增一套“教学笔记内容模型”
   - 把原始笔记拆成可复用的数据结构，而不是直接把长段落硬编码进组件。
   - 至少包含：
     - 关系识别引导：上升 / 太阳 / 月亮的关系深度逻辑
     - 合盘提醒：月亮受克的舒适度风险
     - 上升狮子专题
     - 上升水瓶专题

2. 方法论区新增专题入口
   - 在首页 `Methodology` 区和报告页 `ReadingMethodPanel` 中增加一个默认折叠的专题块。
   - 用户可以先看短摘要，再展开看完整专题。

3. 本命报告新增“上升专题聚焦”
   - 当本命上升为狮子或水瓶时，在报告页增加针对该上升的聚焦卡。
   - 内容需要从“社会化人格面具 / 成长课题 / 情绪与关系提醒 / 资源与行动方式”四个方向压缩呈现。

4. 保留专业占星师可读层
   - 用户默认看到翻译后的现实语言。
   - 专业视角保留技术提醒，例如：
     - 上升与太阳、月亮的关系层级
     - 月亮受克在关系中的优先级
     - 相关宫主星与命主星的典型叙事

### Non-Goals

- 本次不实现完整合盘模块。
- 本次不把所有上升星座一次性补齐，只先支持狮子与水瓶。
- 本次不重写现有本命引擎结论，只增加结构化内容层与专题解释层。

## Expected Outcome

- 首页方法论不再只有抽象原则，而是能让用户快速理解“为什么上升是第一印象，月亮才是亲密层”。
- 报告页在碰到狮子上升、水瓶上升时，能给出更像咨询对话的专题提示。
- 项目获得一个可扩展的“教学资料配置层”，后续可以继续加 `0614/0615` 等专题笔记。
