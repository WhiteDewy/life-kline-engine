## 一、核心原则

### 1.1 宫性大于星性
- **宫位 = 舞台**（这件事发生在哪个领域）
- **行星 = 角色**（谁在演）
- **星座 = 风格**（怎么演）

飞星目录按目标宫位组织（宫性优先）是对的，已验证在实际咨询中更精准。

### 1.2 宫内星 vs 宫主星
- **宫内星 = 过程**。你在做什么、怎么做的、经历了什么。
- **宫主星 = 结果**。这件事最终走向哪里。结果虽然重要，但依赖行运触发。所以很多时候**过程比结果更值得讲**。

### 1.3 古占 vs 现占
- **古占**：这件事容不容易成（事件/结构/得失）
- **现占**：你在心理上怎么处理这件事（动机/感受/演化）

### 1.4 引擎锚点原则
引擎的结构性判断是锚点，不可被推翻。用户反馈"不准"时是取象角度需要切换，不是引擎算错。

### 1.5 钩子原则
每段免费分析末尾种一个好奇心钩子——让用户想继续深入。不卖恐惧，卖好奇心。

## 二、叙事引擎架构

### 2.1 五层叠加模型

```
Base（行星 × 星座）          → 核心行为描述
+ Dignity 修饰               → 能力顺畅/受阻/中性
+ HouseType 修饰             → 公开可见/资源释放/内在消化
+ Aspect 修饰                → 谁在帮/谁在压/和命主的关系
+ Reception 修饰             → 谁给你铺了路
= 完整个人化叙事
```

### 2.2 每层的触发条件

| 层 | 触发条件 | 文件位置 |
|:---|:---|:---|
| Base | 行星 + 星座（始终触发） | `interpretation/narrative_engine.py` |
| Dignity | domicile / exaltation / peregrine / detriment / fall | 同上 |
| HouseType | 角宫(1/4/7/10) / 续宫(2/5/8/11) / 果宫(3/6/9/12) | 同上 |
| Aspect | 被刑冲 / 被拱六合 / 与命主星有相位 | 同上 |
| Reception | 被其他行星接纳（含接纳星等级加权） | 同上 |
